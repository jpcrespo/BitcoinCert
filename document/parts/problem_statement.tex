\section{Problem Statement}

Decentralized solutions for document verification have security problems inherent to the network on which they are deployed.
On Bitcoin, they contain privacy issues or are inefficient in cost and blockspace in solutions using protocols such as OpenTimestamps, Blockcerts, \texttt{OP\_RETURN}, among others.

Taproot applies a new, more efficient, and more private Schnorr digital signature scheme for multisignature security schemes.
It also allows committing multiple scripts into a single output using Merkleized Abstract Syntax Trees (MAST), improving costs and blockspace usage.
This combination has not been previously utilized to create a protocol for document verification.

\subsection{Research Questions}

This work aims to answer:
\begin{itemize}
    \item How are the tools inherent to Taproot leveraged to propose a decentralized verification protocol?
    \item How does anchoring information using Merkle trees within Taproot enhance privacy, scalability, and performance in verification?
    \item Using Taproot, can institutions modify or revoke published certificate?
\end{itemize}

\subsection{Specific Objectives}
To answer the research questions, the following objectives are set:
\begin{enumerate}[label=\alph*)]
    \item To design a Merkle tree containing scripts for verifying the integrity of issued documents.
    \item To design a component that allows the verification of ownership using the digital signature on issued documents.
    \item To design a component that tracks the revocation of a document to verify its validity.
    \item To construct a Proof of Concept (PoC) on a Bitcoin test network.
    \item To estimate the scalability, temporal/spatial complexity, and network costs based on the use of vBytes.
\end{enumerate}
