se busca crear un sistema descentralizado para verificar un certificado digital.

La entidad emisora, por ejemplo  una universidad, genera un certificado digital con toda la información contenida en el archivo. Esta se puede guardar digitalmente pero para el proyecto solo usa el hash de este. Firma digitalmente el archivo con una clave que pertenece a una autoridad. Ancla el hash en un leaf de un arbol de merkle (pudiendo escalar a un numero N de certificados) y obteniendo el merkle root hash construye la dirección taproot usando tweak con un esquema multifirma (como musig) con schnorr. En el opreturn de esta tx publica la public key usada para firmar los certificados.
El emisor provee a un estudiante o usuario el certificado propio y un archivo binario (que puede ser un json) con la información de la firma otorgada, y los hashes para reconstruir el hashroot con la public key usada en la tx de bitcoin y otros datos necesarios como el idtx donde esta contenido.

Un verificador correra un software creado por nosotros que le pedira ambos documentos, el archivo digital y el binario extra.

el software recurre a la tx mediante el idtx, verifica la firma comparando el valor provisto en el binario con el calculo de la firma con el pk y el hash del archivo, si son iguales se verificara que el emisor ha generado ese certificado.
Luego reconstruye con los otros datos y el hash del certificado la direccion taproot y la compara con la que esta en el idtx, si son iguales demustra que el documento presentado es integro y no se ha modificado.

Finalmente el software verifica que la tx no tiene otras tx en su output, si lo tiene recurre a la ultima y verifica si el hash reconstruye un path que daria al certificado como invalido. (aun debo pulir esta parte).

En caso de que la clave que se usa para firmar los certificados sea robada, los que poseen las claves de bitcoin pueden replicar la información publicando una nueva clave publica para verificar la firma, solo dando validez de la firma anterior a esa tx anterior.
