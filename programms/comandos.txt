Iniciamos con el nodo

bitcoind -regtest -daemon   

creamos wallet1

bitcoin-cli -regtest createwallet wallet1

o la cargamos si ya existe

bitcoin-cli -regtest loadwallet wallet1

info
bitcoin-cli -regtest -rpcwallet=wallet1 getwalletinfo


bitcoin-cli -regtest -rpcwallet=wallet1 getnewaddress "funding" bech32

bcrt1qys564cwr75jv908tvrtwyf3znnvnasqnjvh4d6

creamos una address para recibir fondos


Pagamos 50 btc a una direccion generada

bitcoin-cli -regtest generatetoaddress 101 bcrt1qys564cwr75jv908tvrtwyf3znnvnasqnjvh4d6

bitcoin-cli -regtest -rpcwallet=wallet1 getbalances

con eso vemos los btc gastables

GASTAR

la direccion generada en python es:

mr412Reo2o2VeoaBSAcRUNZ6opHjydJWjc

para gastar desde el nodo tenemos 
bitcoin-cli -regtest -rpcwallet=wallet1 sendtoaddress mr412Reo2o2VeoaBSAcRUNZ6opHjydJWjc 10

responde:

bb407d7f1540169fb53c17e647805af5f277a60004ee2e83a882ab76676047b8

esa es la idtx, para que este en un bloque debemos generar un bloque

bitcoin-cli -regtest generatetoaddress 1 bcrt1qys564cwr75jv908tvrtwyf3znnvnasqnjvh4d6


para ver si ya esta generado:
bitcoin-cli -regtest -rpcwallet=wallet1 gettransaction 3f3203de15c9acd65e0a2d679a68a54dab58f9a62a2801ab19455aa54502f7b8



NUEVO TIPO DE GASTO

ahora generamos otra dirección de gasto en el nodo para gastar la tx generada en python

bitcoin-cli -regtest -rpcwallet=wallet1 getnewaddress "funding legacy" legacy

mfamcaEFg5vJiLuMSK7mE7eBQNPrbg54yq

Ahora enviamos de PYTHON a esta nueva address.

Rastreamos el input:

bitcoin-cli -regtest listunspent 1 999999 '["mr412Reo2o2VeoaBSAcRUNZ6opHjydJWjc"]'

-> el anterior comando permite ver las utxo de la wallet que el nodo controla
